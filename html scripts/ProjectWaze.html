Waze User Churn & Retention Analysis

A data-driven case study focused on identifying early warning signs of user churn in the Waze navigation app and translating machine learning insights into actionable retention strategies.

Problem Statement

Waze relies heavily on active users to generate real-time traffic data. User churn directly impacts data quality, network effects, and long-term revenue through location-based advertising.

The objective of this project was to predict user churn using behavioral data and uncover patterns that signal disengagement early enough for intervention.

Dataset Overview

This project uses a synthetic dataset from the Google Advanced Data Analytics Professional Certificate, designed to simulate real-world Waze user behavior.

Records

14,999 users

Features

13 behavioral and engagement variables

Target

User churn (retained vs churned)

Analytical Approach

1. Data Preparation

2. Exploratory Data Analysis

  • Analyzed engagement metrics such as sessions, drives, and activity days
  • Compared retained vs churned user behavior
  • Identified reduced driving activity as a strong churn signal

3. Feature Engineering & Modeling

  • Built and evaluated multiple classification models
  • Applied SMOTE and class weighting to handle imbalance
  • Optimized models using Random Search and threshold tuning

Key Findings

  • Churned users show a clear decline in driving and activity days
  • Long-term users accounted for a large portion of churn
  • Driving inactivity is an early and reliable churn indicator
  • Class imbalance significantly impacts churn prediction performance

Business Impact

The insights from this analysis can help Waze proactively identify users at risk of churn and apply targeted retention strategies such as personalized engagement, rewards, and feature recommendations.

  • Improved retention ‚Üí stronger network effects
  • Higher data quality for navigation accuracy
  • More effective allocation of retention incentives
